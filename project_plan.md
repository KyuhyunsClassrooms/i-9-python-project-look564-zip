# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

**내가 정리해 둔 책 목록을 검색·필터·정렬·통계·추천하고, ★사용자가 말한 "한 문장"에 가장 잘 맞는 최고의 책을 골라 주며, 고른 책의 다운로드 링크까지 받아오는 "책 검색·추천·다운로드 프로그램" (Anna's Archive 회원 API 활용)**

핵심 아이디어는 **"복잡하게 검색하지 말고, 그냥 한 문장만 말하면 최고의 책을 추천받는다"** 이다. 단순한 "추천기" 예시에서 한 걸음 더 나아가, 12권의 책 데이터를 여러 방식으로 처리하고, 자연어 문장으로 추천까지 하는 종합 프로그램으로 잡았다.

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명:

읽고 싶은 책이 여러 권 쌓이면, "EPUB만 보고 싶다", "최신 책부터 보고 싶다", "어떤 책을 먼저 읽을지 골라 달라"처럼 그때그때 다른 요구가 생긴다. 종이 메모로는 이런 검색·정렬·추천을 매번 손으로 하기 번거롭다.
그래서 책 정보를 한 곳(2차원 리스트)에 모아 두고, **메뉴를 골라 검색·형식 필터·최신순 정렬·통계·자동 추천**을 바로 할 수 있게 만들고 싶었다. 또 마음에 드는 책을 고르면 **Anna's Archive 회원 API**로 다운로드 링크까지 받아오도록 해서, "찾기 → 고르기 → 받기"가 한 프로그램 안에서 이어지게 했다.

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 메뉴 번호 (0~7) — 어떤 기능을 쓸지 고르는 값
- 검색어, 형식 문자열 (EPUB / PDF), 다운로드할 책 번호, **추천을 위한 한 문장(자연어)**
- 프로그램이 미리 가진 데이터: `books` 2차원 리스트(책 12권), `.env` 파일의 회원 API 키(`AA_API_KEY`)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- **검색**: 검색어를 소문자로 바꿔 제목·저자에 부분 일치(`in`)하는 행만 모은다.
- **필터**: 형식 열(2번 열)이 입력한 형식과 같은 행만 모은다.
- **정렬**: 출판연도(3번 열) 기준 내림차순(최신순) 정렬한다.
- **통계**: 형식별 권수를 세고(딕셔너리 카운팅), 평점 합으로 평균을 구하고, 연도를 비교해 가장 최신 책을 찾는다.
- **추천(평점)**: `점수 = 평점 × 2 + (출판연도 − 2000) × 0.05` 공식으로 모든 책의 점수를 매겨 최고점 책을 고른다.
- **★문장 추천**: 사용자 문장 안에 각 책의 키워드(6번 열)가 몇 개 들어 있는지 세어, `점수 = 키워드 일치 수 × 3 + 평점 + (연도 − 2000) × 0.02` 로 가장 잘 맞는 책 한 권을 고른다. 키워드가 하나도 안 맞으면 평점이 가장 높은 책으로 추천한다.
- **다운로드**: 책의 md5 해시가 32자리인지 검사한 뒤, 회원 API에 요청해 링크를 받는다(실패 시 원인을 알려 준다).

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 번호가 매겨진 책 목록(제목/저자/형식/연도/평점)
- 통계 결과(총 권수, 형식별 권수, 평균 평점, 최신 책), 추천 결과(책 제목 + 점수)
- **문장 추천 결과**(가장 잘 맞는 책 + 맞은 키워드 이유 + 점수), 다운로드 링크 또는 안내 메시지

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: **`books`**

```python
books = [
    ["1984",       "George Orwell", "EPUB", 2009, 4.7, "md5_here", "디스토피아 감시 자유 ..."],
    ["Steve Jobs", "Walter Isaacson", "EPUB", 2011, 4.6, "md5_here", "혁신 창업 리더십 인생 ..."],
    # ... 총 12권
]
```

### 한 행의 의미

한 행(`books[i]`)은 **책 한 권**의 정보를 모아 놓은 것이다.

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 | 제목(문자열) | "1984" |
| 1번 열 | 저자(문자열) | "George Orwell" |
| 2번 열 | 형식(문자열) | "EPUB" |
| 3번 열 | 출판연도(정수) | 2009 |
| 4번 열 | 평점(실수, 0.0~5.0) | 4.7 |
| 5번 열 | md5 해시(문자열) | "md5_here" → 실제로는 32자리 |
| 6번 열 | 주제 키워드(띄어쓰기로 구분) | "혁신 창업 리더십 인생" |

예) `books[0][0]` → "1984", `books[0][3]` → 2009, `books[6][6]` → "혁신 창업 리더십 ..."

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| `load_api_key()` | `.env`에서 회원 키 읽기 | 없음 | 키 문자열(없으면 "") |
| `search_books(data, keyword)` | 제목·저자로 검색 | 2차원 리스트, 검색어 | 조건에 맞는 새 2차원 리스트 |
| `filter_by_format(data, fmt)` | 형식으로 거르기 | 2차원 리스트, 형식 | 새 2차원 리스트 |
| `sort_by_year(data)` | 최신순 정렬 | 2차원 리스트 | 정렬된 2차원 리스트 |
| `show_statistics(data)` | 통계 계산·출력 | 2차원 리스트 | 화면 출력 |
| `recommend_book(data)` | 평점 점수로 책 추천 | 2차원 리스트 | (책 행, 점수) 또는 None |
| `recommend_by_sentence(data, sentence)` | ★문장으로 최고의 책 추천 | 2차원 리스트, 문장 | (책 행, 점수, 맞은 키워드) 또는 None |
| `print_books(data)` | 목록 보기 좋게 출력 | 2차원 리스트 | 화면 출력 |
| `get_download_link(md5, key)` | API로 링크 받기 | md5, 키 | 링크 또는 오류 문자열 |
| `main()` | 전체 메뉴 흐름 제어 | 없음 | 프로그램 실행 |

총 **10개** 함수. 입력(`load_api_key`/`main`의 input), 처리(`search`/`filter`/`sort`/`statistics`/`recommend_book`/`recommend_by_sentence`/`get_download_link`), 출력(`print_books`)으로 역할을 나눴다.

## 6. 사용할 제어구조

### 조건문을 사용할 부분

내 계획:

- 메뉴 번호에 따라 기능을 나눈다(`if choice == "1" … elif …`).
- 검색어가 제목·저자에 들어 있는지(`if keyword in title or keyword in author`).
- **문장 안에 책 키워드가 들어 있는지**(`if keyword in sentence`), 형식이 일치하는지, 통계·추천에서 점수가 더 높은지 비교한다.
- 다운로드 입력이 숫자인지(`isdigit()`), 번호가 범위 안인지, md5가 32자리인지, API가 성공했는지 검사한다.

### 반복문을 사용할 부분

내 계획:

- 메뉴를 계속 보여 주기 위해 `while True`로 반복한다(0 입력 시 `break`).
- `books`의 모든 행을 `for`로 하나씩 돌며 검색·필터·통계·추천·출력을 처리한다.
- 문장 추천에서는 책의 키워드를 `for`로 하나씩 돌며 문장에 들어 있는지 센다(반복문 안의 반복문).

## 7. 예상되는 예외 상황

1. **검색·필터 결과가 0건일 때** → `print_books`가 `if len(data) == 0:`으로 잡아 "조건에 맞는 책이 없습니다." 출력
2. **잘못된 메뉴 입력**(0~6이 아님) → `main`의 `else`가 "잘못된 입력입니다. 0~6 중에서 고르세요." 출력 후 다시 메뉴로
3. **다운로드 번호가 숫자가 아님** → `if not answer.isdigit()`로 잡아 "숫자를 입력해야 합니다." / **범위 밖 번호** → "취소했습니다."
4. **다운로드 API 단계별 실패** → `get_download_link`가 ① 키 없음 ② md5가 32자리 아님 ③ requests 미설치 ④ 네트워크 오류(`try/except`) ⑤ 서버 응답 코드 비정상 ⑥ JSON 파싱 실패를 각각 다른 메시지로 안내(현재는 md5가 "md5_here"라 ②에서 멈춤)

## 8. AI에게 도움받고 싶은 부분

1. 자유 주제를 "책 검색·추천 + 문장 추천"으로 구체화하는 방향 점검
2. `books` 2차원 리스트의 열 구성(키워드 열 포함)과 함수 분리(입력/처리/출력) 방법
3. 문장과 키워드를 비교해 점수를 매기는 추천 알고리즘 설계
4. 잘못된 입력·결과 없음·API 실패 같은 예외 상황을 빠뜨리지 않고 처리하는 방법
