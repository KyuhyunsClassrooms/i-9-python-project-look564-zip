# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

**예산과 종류에 따라 메뉴를 추천해 주는 "음식 추천 프로그램"**

핵심 아이디어는 **"오늘 가진 돈이나 먹고 싶은 종류만 정하면, 먹을 수 있는 음식을 골라 준다"** 이다. 메뉴가 많을 때 일일이 보지 않고도 조건에 맞는 음식만 빠르게 추천받을 수 있게 했다.

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명:

배가 고픈데 뭘 먹을지 고르기 어렵고, 가진 돈으로 살 수 있는 게 뭔지 한눈에 보이지 않을 때가 많다. 또 "오늘은 음료만", "디저트가 먹고 싶다"처럼 종류로 고르고 싶을 때도 있다.
그래서 음식 정보를 한 곳(2차원 리스트)에 모아 두고, **메뉴를 골라 전체 보기·예산 추천·종류 추천**을 바로 할 수 있게 만들고 싶었다.

---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 메뉴 번호 (0~3) — 어떤 기능을 쓸지 고르는 값
- 예산(원), 음식 종류(식사 / 음료 / 디저트)
- 프로그램이 미리 가진 데이터: `foods` 2차원 리스트(음식 6개)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- **예산 추천**: 가격 열(1번 열)이 예산 이하(`<=`)인 음식만 모은다.
- **종류 추천**: 종류 열(2번 열)이 입력한 종류와 같은(`==`) 음식만 모은다.
- **입력 검사**: 예산이 숫자인지(`isdigit()`) 확인한 뒤 `int()`로 바꾼다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 전체 메뉴 목록(이름/가격/종류/특징)
- 예산·종류 조건에 맞는 음식 목록
- 조건에 맞는 음식이 없으면 안내 메시지

---

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: **`foods`**

```python
foods = [
    ["김밥", 3500, "식사", "가벼움"],
    ["라면", 4000, "식사", "매움"],
    # ... 총 6개
]
```

### 한 행의 의미

한 행(`foods[i]`)은 **음식 한 개**의 정보를 모아 놓은 것이다.

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 | 이름(문자열) | "김밥" |
| 1번 열 | 가격(정수) | 3500 |
| 2번 열 | 종류(문자열) | "식사" |
| 3번 열 | 특징(문자열) | "가벼움" |

예) `foods[0][0]` → "김밥", `foods[0][1]` → 3500, `foods[2][3]` → "든든함"

---

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| `show_all()` | 전체 메뉴 출력 | 없음 | 화면 출력 |
| `recommend_by_budget(budget)` | 예산 안의 음식 추천 | 예산(정수) | 화면 출력 |
| `recommend_by_type(kind)` | 종류로 음식 추천 | 종류(문자열) | 화면 출력 |
| `main()` | 전체 메뉴 흐름 제어 | 없음 | 프로그램 실행 |

총 **4개** 함수. 처리(`recommend_by_budget`/`recommend_by_type`), 출력(`show_all`), 흐름 제어(`main`)로 역할을 나눴다.

## 6. 사용할 제어구조

### 조건문을 사용할 부분

내 계획:

- 메뉴 번호에 따라 기능을 나눈다(`if choice == "1" … elif …`).
- 예산 이하인지 비교한다(`if food[1] <= budget`).
- 종류가 일치하는지 비교한다(`if food[2] == kind`).
- 예산 입력이 숫자인지 검사한다(`if money.isdigit()`).

### 반복문을 사용할 부분

내 계획:

- 메뉴를 계속 보여 주기 위해 `while True`로 반복한다(0 입력 시 `break`).
- `foods`의 모든 행을 `for`로 하나씩 돌며 출력·추천을 처리한다.

## 7. 예상되는 예외 상황

1. **예산·종류에 맞는 음식이 0개일 때** → `found` 변수를 두어 하나도 못 찾으면 "조건에 맞는 음식이 없습니다." 출력
2. **예산에 숫자가 아닌 값 입력** → `if money.isdigit()`로 막고 "숫자만 입력하세요." 출력
3. **잘못된 메뉴 입력**(0~3이 아님) → `main`의 `else`가 "잘못된 입력입니다. 다시 선택하세요." 출력 후 다시 메뉴로
