# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: **책 검색·추천·다운로드 프로그램 (Anna's Archive 회원 API 활용)**

이 프로그램을 만든 이유:
읽고 싶은 책이 쌓이면 "EPUB만 보기", "최신순으로 보기", "뭘 먼저 읽을지 골라 주기"처럼 그때그때 다른 정리가 필요하다. 그런데 매번 메뉴를 고르고 검색어를 치는 것도 번거롭다. 그래서 **"복잡하게 검색하지 말고, 그냥 한 문장만 말하면 최고의 책을 추천받자"** 를 핵심 아이디어로 잡았다. 책 정보를 2차원 리스트에 모아 두고 검색·필터·정렬·통계·추천을 처리하며, **사용자가 말한 한 문장에 가장 잘 맞는 책 한 권**을 골라 준다. 고른 책은 API로 다운로드 링크까지 받게 만들었다.

---

## 2. 문제 분해 및 모델링

### 입력
- 메뉴 번호(0~7), 검색어(제목/저자 일부), 형식(EPUB/PDF), **추천을 위한 한 문장(자연어)**, 다운로드할 책 번호
- 내부 데이터: `books` 2차원 리스트(12권), `.env`의 API 키(`AA_API_KEY`)

### 처리
- 검색(부분 일치), 형식 필터(일치 비교), 최신순 정렬(연도 내림차순), 통계(권수 카운팅·평균·최신 찾기), 평점 추천(점수 계산 후 최댓값), **문장 추천(문장과 키워드 일치 수로 점수 계산)**, 다운로드(md5 검증 후 API 호출)

### 출력
- 번호 매긴 책 목록, 통계 결과, 추천 결과(책+점수), **문장 추천 결과(책+맞은 키워드 이유+점수)**, 다운로드 링크 또는 안내 메시지

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: **`books`** (책 12권)

한 행의 의미: **책 한 권의 정보 묶음** (`books[i]` = i번째 책)

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 제목 | "Steve Jobs" |
| 1번 열 | 저자 | "Walter Isaacson" |
| 2번 열 | 형식 | "EPUB" |
| 3번 열 | 출판연도 | 2011 |
| 4번 열 | 평점(0.0~5.0) | 4.6 |
| 5번 열 | md5 해시(다운로드용) | "md5_here"(→ 실제 32자리) |
| 6번 열 | 주제 키워드(띄어쓰기 구분) | "혁신 창업 리더십 인생 도전 성공" |

이 2차원 리스트를 사용한 이유: 책마다 여러 정보(제목·저자·형식·연도·평점·md5·키워드)를 함께 묶어야 했기 때문이다. 1차원 리스트 여러 개로 나누면 같은 책의 정보가 흩어지지만, 한 행에 한 책을 담으면 `row[0]`(제목), `row[4]`(평점), `row[6]`(키워드)처럼 **행(책)과 열(속성)**로 깔끔하게 접근할 수 있다. 특히 6번 열 키워드는 "문장 추천"의 핵심 재료가 된다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| `load_api_key()` | `.env`에서 키 읽기 | 없음 | 키 문자열("" 가능) |
| `search_books(data, keyword)` | 제목·저자 검색 | 리스트, 검색어 | 새 2차원 리스트 |
| `filter_by_format(data, fmt)` | 형식으로 거르기 | 리스트, 형식 | 새 2차원 리스트 |
| `sort_by_year(data)` | 최신순 정렬 | 리스트 | 정렬된 리스트 |
| `show_statistics(data)` | 통계 계산·출력 | 리스트 | 화면 출력 |
| `recommend_book(data)` | 평점 점수로 추천 | 리스트 | (책, 점수) 또는 None |
| `recommend_by_sentence(data, sentence)` | ★문장으로 최고의 책 추천 | 리스트, 문장 | (책, 점수, 맞은 키워드) 또는 None |
| `print_books(data)` | 목록 출력 | 리스트 | 화면 출력 |
| `get_download_link(md5, key)` | API 링크 받기 | md5, 키 | 링크 또는 오류 문자열 |
| `main()` | 메뉴 흐름 제어 | 없음 | 프로그램 실행 |

총 **10개** 함수.

가장 중요한 함수: **`recommend_by_sentence(data, sentence)`** (177줄)

설명:
이 프로그램의 핵심으로, 사용자가 말한 **한 문장**에 가장 잘 맞는 책 한 권을 골라 준다.
먼저 `if len(data) == 0:`으로 책이 없으면 `None`을 돌려준다(예외 처리).
그다음 `for`로 모든 책을 돌면서, 각 책의 키워드(6번 열)를 `.split()`으로 단어로 나누고(193줄), **또 다른 `for`로 키워드를 하나씩 돌며** `if keyword in sentence`로 그 키워드가 사용자 문장 안에 들어 있는지 센다(195~196줄). 즉 **반복문 안에 반복문**이 들어간 구조다.
점수 공식은 `점수 = 맞은 키워드 수 × 3 + 평점 + (출판연도 − 2000) × 0.02` 로, **주제가 많이 겹칠수록(키워드 일치)** 크게 가점하고 평점과 최신도를 약간 더한다. 점수가 가장 높은 책과 그 점수, 맞은 키워드 목록을 돌려준다.
예) "혁신과 인생을 바꾸는 이야기를 읽고 싶어" → Steve Jobs는 '혁신','인생' 2개가 맞아 `2×3 + 4.6 + (2011−2000)×0.02 = 6 + 4.6 + 0.22 = 10.82`로 1등(실행 결과와 일치). 키워드가 하나도 안 맞으면 자연히 평점이 가장 높은 책이 추천된다(예외 대비).

두 번째로 중요한 함수: **`get_download_link(md5, api_key)`** (216줄)
API를 부르기 전에 잘못된 상황을 **단계별로 미리 걸러 내는** 예외 처리가 핵심이다.
① 키가 없으면(`if not api_key`) 즉시 안내, ② md5가 32자리가 아니면(`if len(md5) != 32`) 안내(지금 책 목록은 "md5_here"라 여기서 멈춘다), ③ `requests`가 없으면 설치 안내, ④ `try/except`로 네트워크 오류를 잡고, ⑤ 응답 코드가 200이 아니거나 JSON이 깨지면 각각 다른 메시지를 낸다. 잘못된 입력을 미리 막아 프로그램이 죽지 않게 하는 것이 목적이다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분
- 메뉴 분기: `main()`의 `if choice == "1" … elif choice == "0"` (269줄~)
- 검색 포함 판단: `search_books`의 `if keyword in title or keyword in author` (112줄)
- **문장 추천의 키워드 일치 판단**: `if keyword in sentence` (196줄)
- 통계 최신 책 비교(149줄), 추천 점수 비교(`if score > best_score`, 171·199줄)
- 입력 검사: `if not answer.isdigit()`(310줄), 번호 범위(314줄), `get_download_link`의 단계별 검사(216~248줄)

### 반복문 사용 부분
- `while True:`로 메뉴를 계속 보여 주고 0이면 `break`로 종료 (254줄)
- `for row in data:`로 책을 하나씩 돌며 검색·필터·통계·추천 (search 109줄, statistics 142줄, 문장추천 192줄)
- **`for keyword in keywords:`로 한 책의 키워드를 하나씩 확인** (195줄) — 반복문 안의 반복문
- `.env`를 한 줄씩 읽는 `for line in f:` (96줄)

### 조건문과 반복문이 함께 작동한 부분
문장 추천이 대표적이다. **바깥 `for`(모든 책) + 안쪽 `for`(그 책의 키워드들) + `if keyword in sentence`(문장에 있나?)** 가 함께 돌면서, 각 책이 문장과 몇 개나 겹치는지 센다. 이렇게 모은 점수를 `if score > best_score`로 비교해 최고의 책을 골라낸다. 즉 **이중 반복 + 조건 판단**이 "문장 추천"이라는 기능을 완성한다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례
입력: 메뉴 6 → 문장 "혁신과 인생을 바꾸는 이야기를 읽고 싶어"
예상 결과: 혁신·인생 키워드를 가진 Steve Jobs가 추천됨
실제 결과: Steve Jobs 추천, 이유로 '혁신, 인생' 표시, 점수 10.82 (일치).

### 테스트 2: 예외 또는 경계 사례
입력: 메뉴 6 → 문장 "으악 아무거나" (어떤 키워드도 안 맞음)
예상 결과: 프로그램이 멈추지 않고, 평점이 가장 높은 책으로 추천
실제 결과: "딱 맞는 주제어는 없어서, 평점이 가장 높은 책으로 추천했어요." 와 함께 Atomic Habits 추천(일치). 또 다운로드(메뉴 7)에서 "abc"처럼 숫자가 아닌 값을 넣으면 "숫자를 입력해야 합니다."가 나오고 다시 메뉴로 돌아갔다.

### 발견한 오류와 수정한 방법
처음에는 다운로드 번호를 바로 `int()`로 바꿔서 "abc"를 넣으면 프로그램이 죽었다. 그래서 `if not answer.isdigit()`로 **숫자인지 먼저 검사**한 뒤 `int()`로 바꾸도록 수정했다. 또 문장 추천에서 키워드가 하나도 안 맞으면 추천이 비어 버릴까 걱정됐는데, 점수에 평점을 더해 두어 **자동으로 평점 최고 책이 추천되도록** 설계해 해결했다.

## 8. AI 활용 내용 요약

AI에게 도움받은 부분: 입력-처리-출력으로 기능을 나누는 큰 틀, 행=책/열=속성의 2차원 리스트 구조, 예외 처리에서 빠뜨리기 쉬운 경우 점검(자세한 기록은 `ai_log.md`).

AI가 제안했지만 내가 수정하거나 사용하지 않은 부분: 예시 주제(음식 추천기)를 책 주제로 바꾸고 기능을 크게 늘렸다.

최종 코드에서 내가 직접 판단하거나 수정한 부분: **문장 추천 알고리즘(키워드 일치 수 × 3 + 평점 + 최신도)과 키워드 열 설계**, 평점 추천 공식, 10개로 나눈 함수 구성, `isdigit()`·범위·`len(md5)!=32`·`try/except` 예외 처리 전체.

## 9. 한계와 개선 방안

현재 프로그램의 한계:
- 문장 추천이 **단순 키워드 포함 검사**라, 키워드에 없는 동의어(예: '연애')나 문맥은 못 잡는다.
- 책 데이터가 코드 안에 12권 고정이라, 책을 추가/삭제하려면 코드를 직접 고쳐야 한다.
- 다운로드는 회원 API 키와 실제 md5(32자리)가 있어야 진짜 링크가 나온다(없으면 안내만).

다시 개선한다면 추가하고 싶은 기능:
- 키워드를 더 늘리거나 비슷한 말(동의어) 사전을 만들어 문장 추천의 정확도 높이기
- 책 추가·삭제 메뉴와 파일 저장(CSV)으로 데이터를 영구 보관하기

## 10. 코드인터뷰 준비

1. **입력-처리-출력은?** 입력: 메뉴 번호·검색어·형식·**한 문장**·책 번호 + books 리스트/API 키 / 처리: 검색·필터·정렬·통계·평점추천·**문장추천**·다운로드 / 출력: 목록·통계·추천 결과·링크(또는 안내).
2. **2차원 리스트의 행과 열은?** 한 행 = 책 한 권, 0~6번 열 = 제목·저자·형식·연도·평점·md5·키워드.
3. **가장 중요한 함수는?** `recommend_by_sentence` — 문장에 책 키워드가 몇 개 들어 있는지 세어(이중 반복문) 점수를 매기고 최고의 책을 추천한다.
4. **조건문·반복문은 어디에?** while로 메뉴 반복, for로 책 순회, **for 안의 for로 키워드 순회**, if로 메뉴 분기·키워드 일치·점수 비교·입력 검사.
5. **AI가 도운 부분과 내가 고친 부분은?** 큰 틀(입력-처리-출력, 함수 분리)은 도움받고, 문장 추천 알고리즘·키워드 열·점수 공식·예외 처리는 내가 직접 설계·수정했다.

### 코드에 쓰인 주요 문법 (내가 직접 설명할 수 있어야 하는 것)

| 문법 | 어디에 | 한 줄 설명 |
|---|---|---|
