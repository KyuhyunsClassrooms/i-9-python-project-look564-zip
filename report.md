# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: **예산·종류로 메뉴를 골라 주는 음식 추천 프로그램**

이 프로그램을 만든 이유:
배가 고픈데 뭘 먹을지 고르기 어렵고, 가진 돈으로 살 수 있는 게 뭔지 한눈에 안 보일 때가 많다. 그래서 **"가진 돈이나 먹고 싶은 종류만 정하면 먹을 수 있는 음식을 골라 준다"** 를 핵심으로 잡았다. 음식 정보를 2차원 리스트에 모아 두고, 예산 이하의 음식이나 같은 종류의 음식만 추려서 추천해 준다.

---

## 2. 문제 분해 및 모델링

### 입력
- 메뉴 번호(0~3), 예산(원), 음식 종류(식사/음료/디저트)
- 내부 데이터: `foods` 2차원 리스트(음식 6개)

### 처리
- 예산 추천(가격 `<=` 예산 비교), 종류 추천(종류 `==` 비교), 입력 검사(`isdigit()` 후 `int()`)

### 출력
- 전체 메뉴 목록, 조건에 맞는 음식 목록, 없을 때 안내 메시지

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: **`foods`** (음식 6개)

한 행의 의미: **음식 한 개의 정보 묶음** (`foods[i]` = i번째 음식)

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 이름 | "돈까스" |
| 1번 열 | 가격 | 8000 |
| 2번 열 | 종류 | "식사" |
| 3번 열 | 특징 | "든든함" |

이 2차원 리스트를 사용한 이유: 음식마다 여러 정보(이름·가격·종류·특징)를 함께 묶어야 했기 때문이다. 1차원 리스트 여러 개로 나누면 같은 음식의 정보가 흩어지지만, 한 행에 한 음식을 담으면 `row[0]`(이름), `row[1]`(가격)처럼 **행(음식)과 열(속성)**으로 깔끔하게 접근할 수 있다.

---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| `show_all()` | 전체 메뉴 출력 | 없음 | 화면 출력 |
| `recommend_by_budget(budget)` | 예산 안의 음식 추천 | 예산(정수) | 화면 출력 |
| `recommend_by_type(kind)` | 종류로 음식 추천 | 종류(문자열) | 화면 출력 |
| `main()` | 메뉴 흐름 제어 | 없음 | 프로그램 실행 |

총 **4개** 함수.

가장 중요한 함수: **`recommend_by_budget(budget)`**

설명:
사용자가 입력한 예산으로 살 수 있는 음식을 골라 준다.
`for`로 모든 음식을 돌면서 `if food[1] <= budget`로 **가격이 예산 이하인지** 비교하고, 맞으면 출력하면서 `found = True`로 표시한다. 반복이 끝났을 때 `if not found`이면 "예산에 맞는 음식이 없습니다."를 출력한다. 즉 **반복문으로 전부 훑고, 조건문으로 추리고, 결과가 없을 때까지 대비**하는 구조다.
예) 예산 4000원 → 김밥(3500)·라면(4000)·아이스티(2500)·아메리카노(3000)가 추천되고, 돈까스(8000)·케이크(6000)는 제외된다.

---

## 5. 조건문과 반복문 활용

### 조건문 사용 부분
- 메뉴 분기: `main()`의 `if choice == "1" … elif choice == "0"`
- 예산 비교: `recommend_by_budget`의 `if food[1] <= budget`
- 종류 비교: `recommend_by_type`의 `if food[2] == kind`
- 결과 없음 처리: `if not found`
- 입력 검사: `if money.isdigit()`

### 반복문 사용 부분
- `while True:`로 메뉴를 계속 보여 주고 0이면 `break`로 종료
- `for food in foods:`로 음식을 하나씩 돌며 출력·추천

### 조건문과 반복문이 함께 작동한 부분
예산 추천이 대표적이다. **`for`(모든 음식) + `if food[1] <= budget`(예산 이하인가?)** 가 함께 돌면서 조건에 맞는 음식만 골라낸다. `found` 변수로 하나도 못 찾은 경우까지 처리해 프로그램이 빈 결과로 끝나지 않게 했다.

## 6. 실행 결과

대표 실행 결과 (python main.py 실제 실행 결과):

```text
메뉴 선택: 2
예산을 입력하세요(원): 4000
[ 4000원으로 먹을 수 있는 음식 ]
 - 김밥 (3500원, 가벼움)
 - 라면 (4000원, 매움)
 - 아이스티 (2500원, 시원함)
 - 아메리카노 (3000원, 쌉쌀함)

메뉴 선택: 3
종류 입력(식사/음료/디저트): 음료
[ '음료' 추천 결과 ]
 - 아이스티 (2500원, 시원함)
 - 아메리카노 (3000원, 쌉쌀함)
```

(전체 출력은 `sample_output.txt` 참고)

결과 분석:
입력한 예산·종류에 따라 추천 결과가 정확히 달라졌다. 예산 4000원으로는 4000원 이하 음식만, '음료' 종류로는 음료만 추천됐다. 예산을 더 낮추거나 종류를 바꾸면 결과가 달라지므로, **사용자 입력에 따라 결과가 변하는** 조건이 충족된다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례
입력: 메뉴 2 → 예산 4000
예상 결과: 4000원 이하 음식 4개가 추천됨
실제 결과: 김밥·라면·아이스티·아메리카노 추천 (일치).

### 테스트 2: 예외 또는 경계 사례
입력: 메뉴 2 → 예산 "abc" (숫자가 아님)
예상 결과: 프로그램이 멈추지 않고 안내 메시지 출력
실제 결과: "숫자만 입력하세요." 출력 후 다시 메뉴로 돌아감 (일치).

### 발견한 오류와 수정한 방법
처음에는 예산을 바로 `int()`로 바꿔서 "abc"를 넣으면 프로그램이 죽었다. 그래서 `if money.isdigit()`로 **숫자인지 먼저 검사**한 뒤 `int()`로 바꾸도록 수정했다. 또 예산이 너무 낮으면 추천이 비어 버려서, `found` 변수로 결과가 없을 때 안내 메시지를 내도록 했다.

## 8. AI 활용 내용 요약

AI에게 도움받은 부분: 입력-처리-출력으로 기능을 나누는 큰 틀, 행=음식/열=속성의 2차원 리스트 구조, 예외 처리에서 빠뜨리기 쉬운 경우 점검(자세한 기록은 `ai_log.md`).

AI가 제안했지만 내가 수정하거나 사용하지 않은 부분: 예시 코드를 그대로 쓰지 않고, 종류 추천 기능과 결과 없음 처리, 예산 입력 검사를 직접 추가했다.

최종 코드에서 내가 직접 판단하거나 수정한 부분: 4개로 나눈 함수 구성, 예산 `<=` 비교·종류 `==` 비교 로직, `isdigit()`·`found` 변수를 이용한 예외 처리 전체.

## 10. 코드인터뷰 준비

1. **입력-처리-출력은?** 입력: 메뉴 번호·예산·종류 + foods 리스트 / 처리: 예산 비교·종류 비교·입력 검사 / 출력: 메뉴 목록·추천 결과·안내 메시지.
2. **2차원 리스트의 행과 열은?** 한 행 = 음식 한 개, 0~3번 열 = 이름·가격·종류·특징.
3. **가장 중요한 함수는?** `recommend_by_budget` — 모든 음식을 돌며 가격이 예산 이하인 것만 골라 추천한다.
4. **조건문·반복문은 어디에?** while로 메뉴 반복, for로 음식 순회, if로 메뉴 분기·예산 비교·종류 비교·입력 검사.
5. **AI가 도운 부분과 내가 고친 부분은?** 큰 틀(입력-처리-출력, 함수 분리)은 도움받고, 종류 추천·결과 없음 처리·예산 입력 검사는 내가 직접 설계·수정했다.

### 코드에 쓰인 주요 문법 (내가 직접 설명할 수 있어야 하는 것)

| 문법 | 어디에 | 한 줄 설명 |
|---|---|---|
| `food[1] <= budget` | recommend_by_budget | 음식 가격이 예산 이하인지 비교한다. |
| `food[2] == kind` | recommend_by_type | 음식 종류가 입력한 종류와 같은지 비교한다. |
| `money.isdigit()` | main | 입력한 값이 숫자로만 되어 있는지 확인한다. |
| `int(money)` | main | 문자열로 받은 예산을 정수로 바꾼다. |
| `found = True / if not found` | 추천 함수 | 조건에 맞는 음식이 하나라도 있었는지 표시하고, 없으면 안내한다. |
| `while True / break` | main | 메뉴를 계속 보여 주다 0을 입력하면 반복을 끝낸다. |
| `for food in foods` | 전체 함수 | 2차원 리스트의 음식을 하나씩 꺼내 처리한다. |
