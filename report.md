# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: **책 검색·추천·다운로드 프로그램 (Anna's Archive 회원 API 활용)**

이 프로그램을 만든 이유:
읽고 싶은 책이 쌓이면 "EPUB만 보기", "최신순으로 보기", "뭘 먼저 읽을지 골라 주기"처럼 그때그때 다른 정리가 필요하다. 이걸 매번 손으로 하기 번거로워서, 책 정보를 2차원 리스트에 모아 두고 메뉴 하나로 **검색·필터·정렬·통계·추천**을 처리하고, 고른 책은 API로 **다운로드 링크**까지 받게 만들었다.

---

## 2. 문제 분해 및 모델링

### 입력
- 메뉴 번호(0~6), 검색어(제목/저자 일부), 형식(EPUB/PDF), 다운로드할 책 번호
- 내부 데이터: `books` 2차원 리스트, `.env`의 API 키(`AA_API_KEY`)

### 처리
- 검색(부분 일치), 형식 필터(일치 비교), 최신순 정렬(연도 내림차순), 통계(권수 카운팅·평균·최신 찾기), 추천(점수 계산 후 최댓값), 다운로드(md5 검증 후 API 호출)

### 출력
- 번호 매긴 책 목록, 통계 결과, 추천 결과(책+점수), 다운로드 링크 또는 안내 메시지

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: **`books`**

한 행의 의미: **책 한 권의 정보 묶음** (`books[i]` = i번째 책)

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 | 제목 | "1984" |
| 1번 열 | 저자 | "George Orwell" |
| 2번 열 | 형식 | "EPUB" |
| 3번 열 | 출판연도 | 2009 |
| 4번 열 | 평점(0.0~5.0) | 4.7 |
| 5번 열 | md5 해시(다운로드용) | "md5_here"(→ 실제 32자리) |

이 2차원 리스트를 사용한 이유: 책마다 여러 정보(제목·저자·형식·연도·평점·md5)를 함께 묶어야 했기 때문이다. 1차원 리스트 여러 개로 나누면 같은 책의 정보가 흩어지지만, 한 행에 한 책을 담으면 `row[0]`, `row[3]`처럼 **행(책)과 열(속성)**로 깔끔하게 접근할 수 있다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| `load_api_key()` | `.env`에서 키 읽기 | 없음 | 키 문자열("" 가능) |
| `search_books(data, keyword)` | 제목·저자 검색 | 리스트, 검색어 | 새 2차원 리스트 |
| `filter_by_format(data, fmt)` | 형식으로 거르기 | 리스트, 형식 | 새 2차원 리스트 |
| `sort_by_year(data)` | 최신순 정렬 | 리스트 | 정렬된 리스트 |
| `show_statistics(data)` | 통계 계산·출력 | 리스트 | 화면 출력 |
| `recommend_book(data)` | 점수로 추천 | 리스트 | (책, 점수) 또는 None |
| `print_books(data)` | 목록 출력 | 리스트 | 화면 출력 |
| `get_download_link(md5, key)` | API 링크 받기 | md5, 키 | 링크 또는 오류 문자열 |
| `main()` | 메뉴 흐름 제어 | 없음 | 프로그램 실행 |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: **`recommend_book(data)`**

설명:
이 함수는 단순 출력이 아니라 **계산과 판정**이 들어간 핵심 알고리즘이다.
먼저 `if len(data) == 0:`으로 추천할 책이 없으면 `None`을 돌려준다(예외 처리).
그다음 `for`로 모든 책을 돌면서 `점수 = 평점 × 2 + (출판연도 − 2000) × 0.05`를 계산한다. 평점을 크게(×2) 반영하되, 최신 책일수록 약간의 가점을 주는 공식이다. 계산한 점수가 지금까지의 최고점(`best_score`)보다 크면 갱신해서, 마지막에는 **점수가 가장 높은 책 한 권과 그 점수**를 돌려준다.
예를 들어 1984는 `4.7×2 + (2009−2000)×0.05 = 9.4 + 0.45 = 9.85`로 가장 높아 추천된다(실행 결과와 일치).

두 번째로 중요한 함수: **`get_download_link(md5, api_key)`**
API를 부르기 전에 잘못된 상황을 **단계별로 미리 걸러 내는** 예외 처리가 핵심이다.
① 키가 없으면(`if not api_key`) 즉시 안내, ② md5가 32자리가 아니면(`if len(md5) != 32`) 안내(지금 책 목록은 "md5_here"라 여기서 멈춘다), ③ `requests`가 없으면 설치 안내, ④ `try/except`로 네트워크 오류를 잡고, ⑤ 응답 코드가 200이 아니거나 JSON이 깨지면 각각 다른 메시지를 낸다. 마지막에 `download_url`이 있으면 링크를, 없으면 서버가 준 `error`를 돌려준다. 잘못된 입력을 미리 막아 프로그램이 죽지 않게 하는 것이 이 함수의 목적이다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분
- 메뉴 분기: `main()`의 `if choice == "1" … elif choice == "0"` (약 208~249줄)
- 검색 포함 판단: `search_books`의 `if keyword in title or keyword in author` (82줄)
- 통계의 최신 책 비교(`if row[3] > newest_row[3]`, 119줄), 추천 점수 비교(`if score > best_score`, 141줄)
- 입력 검사: `if not answer.isdigit()`(233줄), 번호 범위(237줄), `get_download_link`의 5단계 검사 `if len(md5) != 32` 등(159~188줄)

### 반복문 사용 부분
- `while True:`로 메뉴를 계속 보여 주고 0이면 `break`로 종료 (194줄)
- `for row in data:`로 책을 하나씩 돌며 검색·필터·통계·추천 (예: search 79줄, statistics 112줄)
- `.env`를 한 줄씩 읽는 `for line in f:` (66줄)

### 조건문과 반복문이 함께 작동한 부분
검색에서 `for`로 모든 책을 돌며 각 책마다 `if keyword in ...`로 포함 여부를 판단해 결과 리스트에 모은다. 즉 **반복(모든 책 확인) + 조건(검색어 포함?)**이 함께 동작해 "검색"이라는 기능이 완성된다. 추천도 `for`(점수 계산) + `if`(최고점 갱신)가 함께 작동한다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례
입력: 메뉴 2 → 형식 "epub"
예상 결과: EPUB 책 3권만 출력
실제 결과: 1984 / Pride and Prejudice / Frankenstein 3권 출력 (일치). 소문자 "epub"을 넣어도 `.upper()`로 맞춰 비교하므로 잘 걸러졌다.

### 테스트 2: 예외 또는 경계 사례
입력: 메뉴 6 → 다운로드 번호에 "abc"(숫자 아님), 그리고 md5 미설정 상태
예상 결과: 숫자 안내 또는 md5 안내 메시지가 나오고 프로그램이 멈추지 않음
실제 결과: 숫자가 아니면 "숫자를 입력해야 합니다.", 번호가 맞아도 md5가 8자리라 "오류: 올바른 md5 해시가 아닙니다."가 출력되고 다시 메뉴로 돌아갔다(일치).

### 발견한 오류와 수정한 방법
처음에는 다운로드 번호를 바로 `int()`로 바꿔서 "abc"를 넣으면 프로그램이 죽었다. 그래서 `if not answer.isdigit()`로 **숫자인지 먼저 검사**한 뒤 `int()`로 바꾸도록 수정했다. 또 검색 결과가 없을 때 빈 화면이 나와서, `print_books`에 `if len(data) == 0:` 안내를 추가했다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분: 입력-처리-출력으로 기능을 나누는 큰 틀, 행=책/열=속성의 2차원 리스트 구조, 예외 처리에서 빠뜨리기 쉬운 경우 점검(자세한 기록은 `ai_log.md`).

Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분: 예시 주제(음식 추천기)를 책 주제로 바꾸고 기능을 크게 늘렸다. 추천 점수 공식, md5 열 추가, 단계별 오류 메시지는 쓰지 않고 직접 설계했다.

최종 코드에서 내가 직접 판단하거나 수정한 부분: `recommend_book`의 점수 공식, 9개로 나눈 함수 구성, `isdigit()`·범위·`len(md5)!=32`·`try/except` 예외 처리 전체.

## 9. 한계와 개선 방안

현재 프로그램의 한계:
- 책 데이터가 코드 안에 6권 고정이라, 책을 추가/삭제하려면 코드를 직접 고쳐야 한다.
- 다운로드는 회원 API 키와 실제 md5(32자리)가 있어야 진짜 링크가 나온다(없으면 안내만).

다시 개선한다면 추가하고 싶은 기능:
- 책 추가·삭제 메뉴와 파일 저장(CSV)으로 데이터를 영구 보관
- 평점·연도 외에 장르 열을 추가해 장르별 추천, 검색 결과 안에서 다시 정렬하기

## 10. 코드인터뷰 준비

1. **입력-처리-출력은?** 입력: 메뉴 번호·검색어·형식·책 번호 + books 리스트/API 키 / 처리: 검색·필터·정렬·통계·추천·다운로드 / 출력: 목록·통계·추천·링크(또는 안내).
2. **2차원 리스트의 행과 열은?** 한 행 = 책 한 권, 0~5번 열 = 제목·저자·형식·연도·평점·md5.
3. **가장 중요한 함수는?** `recommend_book` — 점수 공식으로 모든 책을 평가해 최고점 책을 추천한다.
4. **조건문·반복문은 어디에?** while로 메뉴 반복, for로 책 순회, if로 메뉴 분기·검색 포함·점수 비교·입력 검사.
5. **Gemini가 도운 부분과 내가 고친 부분은?** 큰 틀(입력-처리-출력, 함수 분리)은 도움받고, 주제·점수 공식·md5 열·예외 처리는 내가 직접 설계·수정했다.

### 코드에 쓰인 주요 문법 (내가 직접 설명할 수 있어야 하는 것)

| 문법 | 어디에 | 한 줄 설명 |
|---|---|---|
| `keyword in title` | search_books (82줄) | 문자열 안에 검색어가 들어 있는지(부분 일치) 확인한다. |
| `.lower()` / `.upper()` | search/filter (78·90줄) | 대소문자를 맞춰서 "epub"으로 쳐도 "EPUB"과 같게 비교한다. |
| `sorted(data, key=lambda row: row[3], reverse=True)` | sort_by_year (99줄) | 모든 책을 `row[3]`(연도)로 비교해 큰 값부터(`reverse=True`) 정렬한다. `lambda`는 "각 행에서 무엇으로 비교할지" 정해 주는 임시 함수다. |
| `format_count = {}` 딕셔너리 카운팅 | show_statistics (108~117줄) | 형식(EPUB/PDF)을 열쇠, 권수를 값으로 센다. 이미 있으면 +1, 없으면 1로 시작한다. |
| `enumerate(data, start=1)` | print_books (152줄) | 목록을 돌면서 1부터 번호를 같이 붙여 준다(`1. 책`, `2. 책`). |
| `round(best_score, 2)` | recommend_book (144줄) | 점수를 소수점 2자리까지 반올림한다(9.85처럼). |
| `result.get('error', 기본값)` | get_download_link (188줄) | 딕셔너리에 'error' 키가 있으면 그 값, 없으면 기본 문자열을 줘서 KeyError(키 없음 오류)를 막는다. |
| `try / except` | load_api_key·get_download_link | 파일·네트워크처럼 실패할 수 있는 코드를 감싸, 오류가 나도 프로그램이 죽지 않고 안내 메시지를 내게 한다. |

> requests 라이브러리는 다운로드(API 호출)에만 쓰며 선생님 허락을 받고 사용했다. 만약 못 쓰면 다운로드 기능만 빠지고 검색·정렬·통계·추천은 그대로 동작한다.
